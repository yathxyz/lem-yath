The target is the configured Evil layer in
`~/proj/nix/computer/portable/dot_config/emacs/lisp/init-evil.el`, with stock Vim
behavior supplied by the flake-pinned upstream `lem-vi-mode` where the Emacs
configuration does not override it.

## Remaining capability gaps

These bindings are intentionally not mapped to unrelated commands:

| Emacs binding / feature | Gap in Lem |
|---|---|
| Completion-local `C-.` | The ncurses input path cannot represent this key distinctly, so the completion popup uses `C-c a` for its action menu. |
| Exact Which-Key presentation | Active-map composition, timing, width-bounded `C-h` paging, docstrings, help, abort, nested undo, and digit arguments are implemented. Lem still uses its native transient separators and echo rendering, omits Which-Key's default replacement rules, and uses a focused top-level binding view instead of a persistent root popup when `u` removes the only prefix key. |
| Full Embark workflow | The dispatcher has typed, extensible providers, exact-chord target cycling, and a focused action set, but there is no act-all, collect/export/live views, arbitrary Embark action-map composition, or richer embark-consult adapters. |
| Exact Avy/Ispell presentation | Navigation, all stock dispatch actions, `?` help, numbered suggestions, keep-once, session acceptance, personal-dictionary insertion, and manual replacement are implemented. Spell correction intentionally uses alphabetic targets and a bounded Lem completion prompt; buffer-local dictionaries, dictionary switching, Flyspell UI, exotic display/syntax geometry, and exact Emacs choices-window presentation remain absent or approximate. |
| Full evil-surround grammar | The pinned default delimiter, editable tag, function, prefix-function, and Visual Block insertion grammar is present, including tag deletion/change, attribute retention, prompt cancellation, and immediate explicit-`>` submission. Mode-local custom pair functions and multi-character block-string editing are not. |
| Full `compile.el` surface | The daily asynchronous path and installed Evil bindings are covered, but Lem intentionally has one active `*compilation*` session rather than renamed/concurrent result buffers. It does not implement prefix-selected Comint compilation, interactive process input, arbitrary `compilation-error-regexp-alist` and source-search customization, remote/TRAMP source resolution, error severity/threshold/follow modes, or every stock save-query action (`SPC`/`DEL` aliases, `C-r`, `C-f`, `M-~`, `RET`, and `C-h`). Its parser is bounded to the six verified diagnostic families above, so this is not a claim of whole-`compile.el` compatibility. |
| Remaining Ibuffer / Evil Collection surface | Arbitrary Emacs-Lisp predicate filters and the less-used other-frame, view-and-eval, shell, eval, and print bulk commands remain absent. Marked-buffer regexp query-replace omits GNU Lisp-evaluated `\,` replacements. |
| Full Evil terminal prompt editing | State transitions, live navigation, paste, submit, Escape routing, literal cwd, and child cleanup are covered. Lem exposes no safe prompt/cursor-to-process editing API, so `evil-collection-vterm` delete/change operators such as `d`, `c`, and `x` are not emulated against the child prompt. |
| Remaining Notmuch composition | The configured text compose/reply/inline-forward/send, protected-source preservation, address completion, bounded local-file MIME attachment loop, stock draft save/postpone/resume lifecycle, and configured received-part preview/save default are present; the remainder of stock Notmuch's richer part-action map remains absent. |
| Workwin cursor geometry | The active terminal profile colors match, but ncurses cannot reproduce the optional graphical profile's two-pixel bar width or hollow visual cursor. |
| Remaining Evil-Org / evil-collection integrations | The native Org subset now provides GNU-style element, sentence, and structural-paragraph motions, configured endpoints, insert/append commands, active object/element/greater/subtree text objects, bounded `d/x/X/< />` repair, all eight region-aware Visual Meta routes across headings/lists/tables/literal lines, planning fields with removal and warning/delay prefixes, Org-style named/relative date input with a terminal calendar, ordinary timestamp workflows including successive-command ranges and CLOCK field adjustment, and source-block edit/commit/abort/save sessions. Footnote-definition children outside the bounded paragraph/affiliated-table/list/recursive-block/drawer model, richer destructive repair, Org syntax outside the bounded parser, unsupported unsafe list/table edge cases, prefixed live Babel-session editing, and integrations for every other Emacs mode remain absent. |

These are implementation gaps, not untested claims. Closing one requires adding
the missing editor capability or a faithful equivalent, followed by a focused
TUI regression.
